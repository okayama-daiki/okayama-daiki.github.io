## TL;DR

Zed（macOS）は、キーバインド（ショートカット）を IME より先に処理します。日本語IMが「ひらがな」のままでも、変換中を除いて Vim のノーマルモード操作（`hjkl`, `dd`, `i`, `A`, `:` など）が IME に吸い取られません。

## はじめに

Zed というエディターが最近のお気に入りです。起動が速く、Vim ライクなキーバインドがネイティブにサポートされています。
一方で Vim といえば、IME に起因する日本語入力との相性の悪さで有名です。
ただ日本語文章を編集するために、我々は IME がオフであることを確認してインサートモードに入り、IME をオンにして日本語を入力し、ノーマルモードに入るために IME をオフにしてという、煩わしい手順を踏まなければなりません。

IME がオンなのかオフなのかを頭の片隅に置いておく必要があり、貴重な脳の認知資源を消費します。IME が ON であることを忘れてノーマルモードで `i` や `A` を押してしまった日には、いつまで経ってもインサートモードに入れずイライラが募るばかりです。これではとても思考の速度には追いつけません[^2]。

この問題に一石を投じるのが、Zed の macOS 版における Vim モードです。IME 系の入力ソースであっても、IME を起動する前にショートカットを発動するようになりました[^1]。これにより、日本語IMがオンのままでも Vim のノーマル操作が IME に吸い取られず、快適に編集作業を行うことができます。

## こんな感じ

![Example](/images/zed/example.gif)

IME がオンのまま、Vim ノーマルモードで `cc` や `I`、`A` などが実行できました。日本語文書の編集であっても IME のオンオフを気にすることなく、Vim の恩恵を享受できます。

なお、インサートモードからノーマルモードに移動する方法ですが、`Ctrl + [` を習得するのがおすすめです（小指が長ければ `Esc` でも）。`jj` などを `<Esc>` にマッピングしている場合は、おそらく IME をオフにする必要があります。

### 検証環境

- OS: macOS（Tahoe 26.2）
- IME: 日本語IM（ライブ変換: オフ）
- Zed 0.222.4

### 設定方法

`~/.config/zed/settings.json` に以下を追記してください。

```json
{
    "vim_mode": true
}
```

## 背景

Zed の公式ブログ[^1] で、macOS では「IME 系の入力ソースでも、IME を起動する前にショートカットを発動する」方針が明言されています。
つまり、IME がオンでも（未確定文字列が出ていない限り）キーバインドが先に処理され、Vim のノーマルコマンドが実行されます。

> For IME-based input sources, we now trigger shortcuts before invoking the IME system.

また、Zed リポジトリの告知 Issue[^3] にもドンピシャな記述がありました。日本語のような IME メニューを持つキーボードでも、入力切り替えなしでノーマルモードが使えるようになる、とのことです。

> This primarily affects vim mode on keyboards with IME menus (like Japanese), where you'll now be able to use vim normal mode without switching input.

## Zed の課題点

現時点で気になっているのは、次の2点です。

1つは `w` の word 移動です。日本語はスペースで単語を区切らないので、単語境界が期待通りにならないことがあります。句読点まで一気に飛んでしまうことがあり、かなり不便です。
もう1つは `f` / `r` のような「次の1文字を受け取る」系のコマンドが、IMEオン時は概ね使えないことです。`f{char}` や `r{char}` を入力しようとしても、`{char}` が IME に吸い取られます。

## まとめ

Zed（macOS）は「IME を起動する前にショートカットを処理する」ようになり、日本語IMがオンのままでも（変換中を除いて）Vim のノーマル操作が吸い取られなくなりました。
これだけで「IME の状態を気にし続ける認知負荷」と「`i` / `A` 事故」が大幅に減ります。日本語を書きつつ Vim 操作もしたい人には、それだけで乗り換える理由になります。

一方で、`f` / `r` の不安定さや日本語文での `w` 移動など、Vim としての未完成な部分も残っています。このあたりが改善されると、より安心して常用できそうです。

## 補遺

日本語入力と Vim の相性問題は昔から定番で、回避策もいろいろあります。macOS なら Karabiner-Elements のようなキーマップツールで `Ctrl + [` を押しやすくしたり、別キーに `Esc` を割り当てたり。

ただ、`jj` のように「文字入力で `<Esc>` を作る」運用は、IME がオンだとそもそも `j` が文字として扱われがちで相性がよくありません。結局「いま IME がオンか」を気にする負担が残りやすい、というのが根っこにあります。

Zed の今回の変更は、そこを根っこから潰しにいくのが嬉しいところです。
